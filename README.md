# **Инструкция по работе с Git**

## Введение

**Git** — это распределённая система контроля версий, предназначенная для отслеживания изменений в файлах и организации совместной работы над проектами.  
Она позволяет:

- сохранять историю изменений
- работать параллельно нескольким разработчикам
- быстро откатываться к предыдущим версиям
- безопасно экспериментировать с кодом

Git является стандартом де-факто в современной разработке программного обеспечения

---

## Основные команды Git

Ниже приведены базовые команды, необходимые для начала работы с Git

### 1. Инициализация локального репозитория

```bash
git init
```
*Создаёт новый локальный Git-репозиторий в текущей директории*

### 2. Получение информации о текущем состоянии репозитория

```bash
git status
```
*Показывает какие файлы изменены, какие готовы к коммиту*

### 3. Добавление файлов к следующему коммиту

```bash
# Добавить конкретный файл
git add readme.html

# Добавить все измененные файлы
git add .

# Добавить все файлы с определенным расширением
git add *.js
```
*Подготавливает изменения для фиксации*

### 4. Создание коммита

```bash
git commit -m "Добавлен README.md файл"
```
*Фиксирует изменения с комментарием*

### 5. Вывод истории коммитов с их хеш-кодами

```bash
git log
```
*Просмотр истории коммитов*
